This document defines the API contract between the Frontend and Backend teams before implementation begins.

# Endpoints

---

# 1. Get Students

## Endpoint

GET /api/v1/students

---

## Purpose

Returns a paginated list of students.

---

## Query Parameters

| Name | Type | Required | Default | Description |
|------|------|----------|----------|-------------|
| page | int | No | 1 | Current page number |
| pageSize | int | No | 10 | Number of records per page |
| search | string | No | - | Search by student name, email or mobile number |
| sortBy | string | No | firstName | Sorting field |
| sortOrder | string | No | asc | asc / desc |

---

## Success Response

HTTP Status

200 OK

Response

```json
{
  "items": [
    {
      "id": "4b3d4a7e-1e79-44e7-a63d-24dbce72a111",
      "firstName": "Abhishek",
      "lastName": "Upadhyaya",
      "email": "abhishek@campusflow.com",
      "mobileNumber": "9876543210",
      "className": "Grade 10",
      "status": "Active"
    }
  ],
  "page": 1,
  "pageSize": 10,
  "totalRecords": 145,
  "totalPages": 15
}
```

---

## Error Responses

| Status | Description |
|---------|-------------|
|400|Invalid query parameters|
|500|Internal server error|

---

## Business Rules

- Default page size is 10.
- Maximum page size is 100.
- Default sorting is First Name (Ascending).
- Search should support First Name, Last Name, Email and Mobile Number.

---

# 2. Get Student By Id

## Endpoint

GET /api/v1/students/{id}

---

## Purpose

Returns complete information for a specific student.

---

## Path Parameters

| Name | Type | Required | Description |
|------|------|----------|-------------|
| id | Guid | Yes | Student Identifier |

---

## Success Response

HTTP Status

200 OK

```json
{
  "id": "4b3d4a7e-1e79-44e7-a63d-24dbce72a111",
  "firstName": "Abhishek",
  "lastName": "Upadhyaya",
  "email": "abhishek@campusflow.com",
  "mobileNumber": "9876543210",
  "gender": "Male",
  "dateOfBirth": "1990-10-15",
  "className": "Grade 10",
  "address": "Noida",
  "status": "Active"
}
```

---

## Error Responses

| Status | Description |
|---------|-------------|
|404|Student not found|
|500|Internal server error|

---

## Business Rules

- Student Id must be a valid Guid.
- Archived students are not returned.

---

# 3. Create Student

## Endpoint

POST /api/v1/students

---

## Purpose

Creates a new student.

---

## Request

```json
{
  "firstName": "Abhishek",
  "lastName": "Upadhyaya",
  "email": "abhishek@campusflow.com",
  "mobileNumber": "9876543210",
  "gender": "Male",
  "dateOfBirth": "1990-10-15",
  "className": "Grade 10",
  "address": "Noida"
}
```

---

## Validation Rules

| Field | Validation |
|---------|------------|
|First Name|Required, 2-100 Characters|
|Last Name|Required, 2-100 Characters|
|Email|Required, Valid Email|
|Mobile Number|Required, 10 Digits|
|Gender|Required|
|Date Of Birth|Required|
|Class Name|Required|
|Address|Optional, Maximum 500 Characters|

---

## Success Response

HTTP Status

201 Created

```json
{
  "id": "4b3d4a7e-1e79-44e7-a63d-24dbce72a111"
}
```

---

## Error Responses

| Status | Description |
|---------|-------------|
|400|Validation failed|
|409|Email or Mobile Number already exists|
|500|Internal server error|

---

## Business Rules

- Student Id is generated by the server.
- Email must be unique.
- Mobile Number must be unique.

---

# 4. Update Student

## Endpoint

PUT /api/v1/students/{id}

---

## Purpose

Updates an existing student.

---

## Request

Same as Create Student Request.

---

## Success Response

HTTP Status

200 OK

---

## Error Responses

| Status | Description |
|---------|-------------|
|400|Validation failed|
|404|Student not found|
|409|Duplicate Email or Mobile Number|
|500|Internal server error|

---

## Business Rules

- Student Id cannot be modified.
- Archived students cannot be updated.

---

# 5. Delete Student

## Endpoint

DELETE /api/v1/students/{id}

---

## Purpose

Archives a student.

---

## Success Response

HTTP Status

204 No Content

---

## Error Responses

| Status | Description |
|---------|-------------|
|404|Student not found|
|500|Internal server error|

---

## Business Rules

- Student records are archived (Soft Delete).
- Data is never permanently deleted.
- Archived students are excluded from normal searches.
